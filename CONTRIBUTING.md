# Contributing to OpenCell Thermal-Mechanical Energy System

Thank you for your interest in contributing to the OpenCell Project! 🌍

Our mission is to provide **zero-electric cooling and direct mechanical power** to rural communities and farmers worldwide. We believe in open collaboration, rigorous physics validation, and accessible engineering.

## 🤝 How to Contribute

### 1. Reporting Issues & Bugs
If you find a bug in the documentation, a flaw in the physics model, or an error in the build guide:
- Open an **Issue** on GitHub.
- Include:
  - **Environment:** (e.g., "Testing in clay soil, 10ft depth")
  - **Steps to Reproduce:** (e.g., "Mixed aerocement with 20% less water...")
  - **Expected vs. Actual Results**
  - **Photos/Sketches** (if applicable)

### 2. Suggesting Improvements
Have an idea to improve the efficiency, lower the cost, or adapt the system for a different climate?
- Open an **Issue** labeled `enhancement`.
- Describe the proposed change and the **physics rationale** behind it.
- We prioritize changes that:
  - Increase thermal efficiency (ΔT).
  - Reduce material costs.
  - Simplify construction for low-tech environments.

### 3. Submitting Code & Scripts
We welcome scripts for:
- **Calculations:** Python/Bash scripts for BTU, airflow, or water yield.
- **Automation:** Tools for data logging or IPFS deployment.
- **Simulation:** Models for thermal dynamics (e.g., COMSOL, ANSYS exports).

**Guidelines:**
- Use **Markdown** for documentation.
- Use **Python 3** or **Bash** for scripts.
- Comment your code clearly.
- Include a `README` snippet explaining how to run it.

### 4. Sharing Field Data & Case Studies
This is the **most valuable contribution**!
- Did you build a prototype? **Share your results.**
- Did you measure the cooling output? **Upload the data.**
- Did you adapt the formula for local materials? **Document the changes.**

Submit your findings via:
- **GitHub Issues** (for quick updates).
- **Pull Requests** (for formal documentation updates).
- **Email** (for raw data files) jrm8908@proton.me

## 🧪 Validation & Peer Review

We prioritize **scientific rigor**. Before a new claim is added to the official documentation:
1. **Theoretical Review:** Does it violate thermodynamics?
2. **Experimental Verification:** Has it been tested in the field?
3. **Peer Review:** Has it been reviewed by at least one other contributor?

## 🛠️ Code of Conduct

We are committed to a welcoming and respectful environment.
- **Be Respectful:** Treat all contributors with dignity.
- **Be Constructive:** Critique ideas, not people.
- **Be Inclusive:** We welcome builders from all backgrounds, skill levels, and locations.

## 🚀 Getting Started

1. **Fork** this repository.
2. **Clone** it to your local machine.
3. **Create a Branch** for your work: `git checkout -b feature/my-improvement`.
4. **Make your changes** and test them.
5. **Commit** with a clear message.
6. **Push** and open a **Pull Request**.
